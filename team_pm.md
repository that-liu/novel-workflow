# NovelCraft 产品改进需求清单

> 分析日期: 2026-05-24
> 技术栈: Next.js 16 + TypeScript + Tailwind CSS + DeepSeek v4-pro API
> 分析范围: 全部 10 个页面、7 个核心组件、5 个 API 路由、数据模型

---

## P0（必须修）— 严重阻碍使用

### 1. 数据存储碎片化：世界观和时间线存在 localStorage

世界观设定（`world_${id}`）和时间线事件（`timeline_${id}`）存储在浏览器 localStorage，而小说主体数据（角色、章节、大纲）存储在服务端 JSON 文件。这导致：
- 用户清除浏览器缓存后，世界观和时间线数据永久丢失
- 故事圣经页面（`bible/page.tsx` 第 19 行）读取 localStorage 拼装视图，无法保证数据完整
- 跨设备使用时完全无法同步

**建议：** 将 `WorldSetting` 和 `TimelineEvent` 并入 `Novel` 数据模型，统一走服务端存储 API。

### 2. 章节编辑器无自动保存

写作页面（`ChapterEditor.tsx`）仅在用户按 `Ctrl+S` 或修改章节状态时才触发保存。若浏览器崩溃、误关标签页，所有未保存内容丢失。
- 第 30-34 行的 `keydown` 监听是唯一保存路径
- 无 `content` 变化的 `useEffect` 防抖自动保存
- 无"未保存变更"视觉指示器

**建议：** 内容变更后 3 秒防抖自动保存，并在编辑器顶部显示保存状态（已保存/保存中/未保存）。

### 3. 删除操作无确认弹窗

- `ProjectCard.tsx` 第 18 行：点击 ✕ 直接删除项目，无二次确认
- 角色页面（`characters/page.tsx` 第 72 行）：删除角色无确认
- 大纲页面（`outline/page.tsx` 第 122 行）：删除章节无确认

**建议：** 所有 destructive 操作前弹出确认对话框，支持撤销的短期 Toast 通知。

### 4. AI 错误被静默吞掉

多处代码使用 `catch { /* ignore */ }` 模式：
- `ChapterEditor.tsx` 第 62 行
- `CharacterForm.tsx` 第 36 行
- `outline/page.tsx` 第 60 行

当 AI API 返回错误时，用户看不到任何提示，只发现生成按钮恢复但无结果。

**建议：** 所有 AI 调用 catch 块统一显示用户可见的错误 Toast，并在连续失败时降级提示。

## P1（应该做）— 明显提升体验

### 1. 写作页缺少上下文面板

写作者在编辑器中无法看到角色列表、世界观摘要和故事时间线，必须切换到其他标签页查看。`novelContext` 构造了上下文但只传给 AI，未展示给用户。

**建议：** 编辑器右侧增加折叠式上下文面板，显示当前可用角色、世界观摘要、当前章节上下文的引用章节。

### 2. 快速创建完成后不支持编辑再保存

`QuickCreate.tsx` 的 AI 一键生成流程结束后，用户只能选择"保存"或"放弃重来"，无法在保存前修改 AI 生成的书名、简介、角色或大纲。

**建议：** 生成完成后进入一个编辑预览状态，允许用户自由修改元信息、角色和大纲后再确认保存。

### 3. 项目仪表盘缺少搜索与筛选

`page.tsx` 中以网格平铺所有项目，没有搜索框、类型筛选或排序切换。项目数量增多后难以定位。

**建议：** 增加搜索框（按标题/简介关键词）、类型下拉筛选、排序（最近更新/创建时间/字数）。

### 4. 导出格式太有限

只支持 Markdown、TXT、HTML 三种格式。网络小说常见的 EPUB（电子书标准格式）、PDF、DOCX 均不支持。

**建议：** 至少增加 EPUB 格式导出（可使用 `epub-gen` 或类似库），这是发布到阅读器的最常用格式。

### 5. 故事圣经数据不一致

`bible/page.tsx` 从 localStorage 读取时间线数据（第 19 行）和从 novel.notes 解析世界观数据（第 18 行），解析逻辑脆弱，且生成时机不明确。世界观修改后，"保存全部"按钮将世界观写入 `novel.notes`，但时间线从未被写入服务端。

**建议：** 同 P0-1，统一数据模型后直接读取 Novel 对象渲染，消除不一致。

### 6. 时间线与章节无双向同步

时间线事件可以选择关联章节（`timeline/page.tsx` 第 88-97 行），但删除章节后关联时间线事件变成孤儿记录，没有清理或提示。

**建议：** 删除章节时扫描并更新关联的时间线事件；在时间线页面用链接（而非 select）指向章节，点击跳转到写作页。

### 7. 一键成书（QuickCreate）无进度重试

`QuickCreate.tsx` 四阶段分步生成，但任一步骤失败（如 JSON 解析出错）则整个流程中止，不会跳过失败步骤继续。

**建议：** 单个阶段失败后，使用默认值填充该阶段数据，继续后续阶段。最终在预览中让用户手动修改失败的部分。

### 8. 无项目级字数目标追踪

仪表盘创建表单中"目标字数"输入框是 `disabled` 状态（`page.tsx` 第 116 行），目前只能按章节设置字数目标，缺少项目整体规划。

**建议：** 启用目标字数字段，在项目概览和仪表盘上显示达标进度条，支持按目标拆分到各章节的计划字数。

### 9. 头脑风暴对话不可保存导出

`brainstorm/page.tsx` 中 AI 对话记录仅在页面内存中保留，刷新即丢失。有价值的创意讨论无法归档到项目笔记中。

**建议：** 增加一键"将对话导出到创作笔记"功能；或将对话历史保存在项目数据中（可折叠收起）。

### 10. 写作页面章节切换不够直接

写作页的章节切换器使用按钮式标签（`write/page.tsx` 第 51-63 行），章节很多时会换行混乱，且没有搜索或跳转功能。

**建议：** 改为下拉选择器 + 前后章节导航按钮（< 上一章 / 下一章 >），更适合大量章节场景。

## P2（可以做）— 锦上添花

1. **暗色模式**：写作工具用户通常长时间使用，暗色模式可显著降低视觉疲劳
2. **写作计时器 / 番茄钟**：集成专注写作计时，统计有效写作时间
3. **每日写作目标与连续写作记录**：如"今日已写 2000/5000 字"、"连续写作 7 天"
4. **AI 配图生成**：根据场景或角色描述，调用 AI 生成立绘/插画
5. **情节漏洞检测**：一键分析整本书的逻辑一致性、角色行为合理性
6. **模板市场**：预置各类型（玄幻/言情/悬疑等）的小说结构模板
7. **发布集成**：一键导出到起点、晋江等平台的格式
8. **阅读统计**：预估阅读时长、章节字数分布柱状图、写作趋势图
9. **故事圣经分享链接**：生成公开只读链接，方便发给编辑或 beta 读者
10. **多语言 / 国际化**：当前纯中文 UI，可考虑支持英文等更多语言
11. **协作批注**：邀请协作者在章节上做批注和修订建议
12. **全屏写作模式**：最小化 UI 干扰，专注创作

## 用户故事

### 故事 1：自动保存防丢失

> **As a** 正在创作长篇小说、经常忘记手动保存的用户  
> **I want to** 编辑器每隔几秒自动保存我的写作内容，并在标题栏显示保存状态  
> **So that** 我不用担心浏览器崩溃或误关标签页导致几小时的创作白费

### 故事 2：统一数据不丢失

> **As a** 在多个设备（笔记本、台式机、平板）上写作的用户  
> **I want to** 世界观设定和时间线事件和小说其他数据存在一起，跨设备同步  
> **So that** 我无论在哪个设备上打开项目，都能看到完整的设定，不会因为清缓存而丢失数据

### 故事 3：导出到电子书阅读器

> **As a** 完成了一部小说准备投稿和自出版的作者  
> **I want to** 一键导出 EPUB 格式，包含封面、目录和样式排版  
> **So that** 我可以在手机/ Kindle 上直接阅读效果，或者直接投递给出版社和网络平台

# 用户反馈：NovelCraft 从作者视角的体验报告

> 我是写网文的。模拟了一遍完整流程：首页 → 创建 → 世界观 → 头脑风暴 → 角色 → 时间线 → 大纲 → 写作 → 圣经 → 导出。以下是心里话。

---

## 一、整体感受：方向对了，但糙

看得出想做一个"AI 辅助的完整写作工作流"，路线图是清晰的。但落到每个页面上，感觉像 MVP 直接上线了——功能都有了，但每一个都差那么一口气，用起来总觉得"差点意思"。

---

## 二、逐页吐槽

### 1. 首页 / 项目列表

**好的一面：**
- 项目卡片 + 进度条 + "继续写作"快捷入口，这个设计不错
- "一句话成书"快速生成完整项目，很酷的卖点

**痛点：**

- **"目标字数"输入框是 disabled 的**，那放在那里干嘛？勾引我但又不让我用。要么删了，要么真正支持设定每本书的字数目标并在进度条体现。
- **创建项目后不会跳转**。我点"创建项目"→表单消失→我还是在首页列表。然后我得手动找到新项目点进去。应该创建成功后直接跳到 `/project/{id}`。
- **QuickCreate 没有中途取消能力**。它跑四个阶段（meta→world→characters→outline），每一步几十秒。如果第三步发现方向不对，没办法取消重来，只能等全部跑完或者刷新页面丢数据。
- **QuickCreate 生成的章节标题太泛**。试了几次，出来的标题全是"新的开始""命运的转折""真相大白"这种套话，缺少类型特色。跑完四阶段大概要 2-3 分钟，等这么久就拿到这种大纲，容易失望。

### 2. 项目概览

- **统计卡片：进度按 "done" 章节算**。我写了个开头标了 writing，进度 0%。哪怕写了 5000 字也是 0%。建议按字数比例或者引入一个更细的进度模型。
- **八个入口卡片没有完成状态标记**。我设完了世界观回来，它和没设过长得一模一样。我需要知道哪步做过了、哪步还没碰。
- **没法标记项目状态**。开了三个坑，哪个在更、哪个在屯稿、哪个弃坑了，首页完全看不出来。

### 3. 世界观设定

- **AI 聊天和表单是割裂的**。我在左边填了两段设定，然后在右边的 AI 对话框问"帮我补充地理设定"，AI 不知道我刚才写了什么。它的 system prompt 只包含 `novel.notes`，但我刚刚打在文本框里的内容还没保存，AI 看不见。要达成"AI 理解我写了什么"的效果，我得先点保存再切到 AI 聊天，多了一步。
- **六个字段不够灵活**。我的故事需要"科技树""货币系统""宗教信仰"这些自定义维度。现在只能塞进 notes 或者硬挤到已有的某个字段里。
- **"保存全部"按钮在右上角，太容易忘记**。填了五段之后点 AI 聊天去了，再切回来发现刚才填的没存。应该加个自动保存（像写作页那样）。
- **每段文本框固定 96px 高**，写长一点就变成小 scroll，非常难受。

### 4. 头脑风暴

- **左侧笔记是手动保存的**。这和世界观的问题一样——没有自动保存，切换页面/刷新就丢。既然写作页都有 debounce 自动保存，为什么这里没有？
- **AI 对话记忆不跨页面**。我在这里和 AI 聊了 20 分钟，觉得差不多了，去角色页设了个人物，再回到头脑风暴，之前的对话记录全没了。那长长的一大段思考就蒸发了。至少应该把对话记录持久化。
- **"建议提问"写死在 UI 里**。这些提示是固定的，不会根据我已经写过的东西动态变化。要是我已经聊过开篇了，它还显示"帮我想几个开篇方式"。应该根据对话历史动态生成建议。

### 5. 角色设计

- **AI 生成角色是一次性全部填充**。点了"AI 生成全部"，它把 personality/backstory/motivation 全写了，但我可能只需要它帮我写个背景故事。应该每行都有单独的小 AI 按钮。
- **角色没有关联关系**。A 和 B 是什么关系？同盟、敌对、恋人、师徒？完全没地方记录。这对写小说超重要。
- **角色一多就难找**。十几个角色后，没有搜索、没有按角色类型（主角/反派/配角）筛选。
- **角色卡片显示的内容有限**。卡片只展示了 personality 前两行和 motivation 摘要。要是我想快速对比两个角色的背景故事，要点进去一个个看，没有并排对比。
- **删除角色有确认弹窗，好评。但没有撤销**。手滑点错就没了。

### 6. 故事时间线

- **"时间"字段是纯文本输入**。没有任何格式提示或日期选择器。混乱的输入方式导致时间线很难读——有的写"三年前"，有的写"公元 2345 年 3 月"，有的写"第 5 章之后"。
- **没有真正的可视化时间线**。现在是一个垂直列表，和普通的 todo list 差不多。我期待的是一个横向/纵向的时间轴视觉展示，事件之间按比例显示间隔。
- **事件关联章节时，下拉菜单只显示已有章节的 ID**。如果我先做时间线再做大纲（完全合理的顺序），章节下拉就是空的。流程耦合太紧。
- **大量事件时列表太冗长**。10 个事件以上就要不断滚，应该支持折叠/展开。

### 7. 情节大纲

- **拖拽排序体验一般**。拖拽时的视觉反馈只有 opacity 变化，没有占位指示线，不知道松手后插到哪。而且移动端完全不能用。
- **AI 生成大纲是在末尾追加**。如果我已经手写了几章，点 AI 生成会在末尾再加 8 章。它不会问我"你是想补充还是替换"。大概率我想要的是替换或者插入而不是无脑追加。
- **大纲卡片上字数统计来自 chapter.wordCount**，但这个值只在写作页更新。如果我只在大纲页写了标题和摘要就切走了，字数永远是 0。应该在前端实时计算。
- **"写作"按钮链接到写作文本页**——这个设计很好，但缺少一个"批量展开所有摘要"的功能，我想一口气看完整个故事脉络。

### 8. 写作（核心痛点集中营）

这是最重要的页面，所以槽点最多：

- **纯文本框写作，没有富文本/预览模式**。我写网文的，想看到粗体、斜体、分段渲染效果，现在就是白底黑字一个大框。起码要给个"预览" tab 吧？
- **AI 七种模式功能好，但用起来心惊胆战**：
  - **"润色"直接替换全文**。我写了 3000 字，点了润色，AI 给我换了 2000 字，我原来的内容去哪了？没有版本历史，没有撤销，Ctrl+Z 不一定管用（textarea 的 undo 被 AI 改写打断）。
  - **"续写"在文末追加**，但如果我光标在中间，我想从中间续写呢？现在是把 AI 生成的内容贴在文末，然后我得手动剪切粘贴到想放的位置。
  - **审稿输出也贴在正文里**。审稿建议应该单独显示在侧边栏或弹窗里，而不是混入正文，我还得手动删。
- **字数目标（wordGoal）是全局写死的 2000**。每章的字数目标应该可以单独设定。有的章我打算写 5000 字高潮戏，有的过渡章 800 字就够了。
- **右侧 AI 面板没有上下文感知**。我选中了一段文字，想让它"优化这段"，它不知道我选了哪段，优化的是全文前 2000 字。应该支持选中文字后发送给 AI。
- **没有快捷键优化**。写作者恨不得双手不离键盘。现在除了 Ctrl+S 保存和 Ctrl+Enter 发送指令，其他的都要用鼠标去点侧边栏。应该有类似 /continue、/improve 这样的斜杠命令直接在编辑器里触发。
- **自动保存的反馈不够明显**。右上角小小的"已保存"文字，写作时根本不会注意到。建议在标题栏或状态栏加更明显的保存指示。
- **切换章节时没有确认**。如果写了半天没保存，点了上面另一个章节的标签，内容直接丢了。（其实是 auto-save 3 秒 debounce，但如果打字停了 2 秒就切页面，确实会丢。）

### 9. 故事圣经

- **纯只读预览，不能编辑**。我在圣经里发现世界观有个矛盾（比如第三章说魔法需要吟唱，第五章说可以默发），我想改，只能回到世界观页面或者写作页面改，不能就地修改。
- **缺少交叉引用**。圣经是"把所有信息堆在一起"，但没有"角色 X 在第 3、7、12 章出现"这样的交叉索引。
- **打印功能基础**。直接调用 `window.print()`，打印出来的样式没有专门优化，背景色、卡片阴影在打印时很难看。
- **没有导出圣经本身的功能**。我想把圣经打印出来做桌面参考，但现在的打印效果不行。应该支持导出格式化的 PDF 或 Markdown 版本。

### 10. 导出

- **没有 EPUB！没有 EPUB！没有 EPUB！** 这是最大的缺失。网文作者要上架、要给编辑看、要发朋友试读，EPUB 是标准格式。Markdown 和 TXT 对非技术读者不友好，HTML 在阅读器上体验差。
- **没有 PDF 导出**。同上，给编辑/出版社看的时候 PDF 是刚需。
- **导出功能没有自定义选项**。不能选择导出范围（仅已完成章节还是全部），不能选择是否包含摘要，不能自定义文件命名规则。
- **预览区域是纯文本**。和写作页一样没有富文本渲染。看预览和看编辑器的感觉差不多，没有"成品感"。
- **"复制全文"复制的是 TXT 格式**。应该让用户选择复制的格式。

## 三、结构性缺失（全局问题）

### 3.1 AI 能力的问题

- **AI 对话不持久化**：这是最让我难受的。我在每个页面和 AI 聊的内容都记不住。头脑风暴聊的 idea，到了角色页 AI 就"失忆"了。应该在全局维护一个"项目级 AI 记忆"，让 AI 知道整个项目的上下文。
- **AI 模型选择藏得太深**：在 AI Chat 组件里有个很小的下拉选模型，但 `callAI`（用于非流式请求如 AI 生成大纲、角色生成）全部硬编码了默认模型，用户没法选。如果我觉得 R1 推理效果好，在角色生成页也用不了。
- **没有 AI 写作风格配置**：AI 写出来的东西风格是随机的。我不能告诉它"我写的是唐家三少那种爽文风"或者"我要余华那种冷峻叙事"。每次都要在 prompt 里暗示很累。
- **没有批量 AI 操作**：比如"AI 帮我给所有草稿章节生成摘要"，现在得一章一章点。

### 3.2 数据安全

- **所有数据存在本地 JSON 文件**：`data/*.json`，没有云备份。电脑坏了、重装了、文件夹误删了，小说就全没了。作为一个写了三个月的人，我会焦虑。
- **没有手动备份/导出全部项目的功能**：我至少想每周打包下载一次所有项目。
- **没有版本历史**：修改就覆盖了，我永远找不回三天前那个更好的版本。

### 3.3 编辑器基础体验

- **没有字数统计面板**：写了多少字、今天写了多少、连续写了几天的 streak、平均速度，这些写作者在意的数据完全缺失。
- **没有全站搜索**：我想搜"魔法"这个词在所有章节里出现了几次，做不到。
- **没有阅读模式**：写了之后想从头到尾通读一遍找感觉，现在的界面是编辑态，到处都是按钮和输入框，没法沉浸阅读。
- **没有注释/批注系统**：想在某段旁边写"这里后面要呼应前文"，做不到。

### 3.4 界面与交互

- **整体设计偏灰白冷淡**：写小说是创造世界的事情，UI 可以更有温度和氛围感。不要求花哨，但现在的设计太像后台管理系统。
- **缺少移动端适配**：侧边栏布局、编辑器双栏在手机上完全没法用。虽然写小说主要在电脑上，但灵感来了掏出手机记一段是刚需。
- **没有暗色模式**：写到凌晨两点，白底黑字晃眼睛。这对写作者是刚需。
- **加载状态只有 "加载中..." 三个字**：从 API 拿数据时，整个页面就是一行字。至少应该 skeleton loading。

## 四、和小说的竞品对比缺了什么

| 功能 | NovelCraft | 竞品（如 yWriter, Scrivener, 纯纯写作） |
|------|-----------|--------------------------------------|
| 全量备份/导出 | 手动复制 JSON 文件 | 一键备份 + 云同步 |
| 场景/章节分拆 | 只有章节 | 场景 + 章节 + 幕三层结构 |
| 角色关系图 | 无 | 有 |
| 写作目标/番茄钟 | 无 | 有 |
| 卡片/大纲视图 | 列表 | 卡片墙 + 大纲树 + 思维导图 |
| 自定义元数据 | 不可 | 可添加自定义字段 |
| 写作统计 | 无 | 每日/每周/每月统计 |
| 批注/修订 | 无 | 有 |
| 拼写/语法检查 | 无 | 有（中文需要特殊处理） |
| 导出格式 | MD/TXT/HTML | EPUB/PDF/DOCX/LaTeX |

---

## 五、如果只能改三个地方

排名分先后：

1. **写作编辑器重做**：支持预览模式、AI 建议不混入正文、选中文字后 AI 操作、撤销/版本历史。这是作者花时间最多的地方，体验必须做到 90 分。
2. **EPUB/PDF 导出**：写完了一部小说，给朋友看的时候发过去一个 .txt 文件，人家会觉得你是业余的。EPUB 是网文界流通货币，不能没有。
3. **AI 记忆跨页面持久化 + 项目级上下文**：AI 现在是每个页面一个"临时工"，聊完就忘。应该改成 AI 知道整个项目的世界观、角色、大纲、写作进度，做任何页面的 AI 辅助时都能引用全局信息。这才能体现"AI 写作工作流"的核心理念。

---

以上是用了一轮之后的真实感受。产品的骨架是好的，方向和模块划分没问题，但每一块都需要打磨到"让作者觉得舒服"的程度，而不是"功能实现了就行"。
